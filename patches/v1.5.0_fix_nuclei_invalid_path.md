# Patch v1.5.0 — Fix: Nuclei Patterns "Invalid Path" Error

## Date
2026-07-05

## Symptom
Settings → Tool Settings → Nuclei tab showed:
```
Currently available custom Nuclei templates
Error!
Invalid Path
```

## Root Cause Analysis

**File**: `scanEngine/templates/scanEngine/settings/tool.html` — line 105

The Django template used `{{ file.3 }}` to extract the template filename from the full path.

The path split logic:
```
Path: /home/rengine/nuclei-templates/ssrf_nagli.yaml
Split: ['', 'home', 'rengine', 'nuclei-templates', 'ssrf_nagli.yaml']
Index:   0     1        2              3                    4
```

`file.3` = `"nuclei-templates"` (directory name) — **WRONG**
`file.4` = `"ssrf_nagli.yaml"` (actual filename) — **CORRECT**

This caused every button to call `load_nuclei_template('nuclei-templates')`, which
tried to open `/home/rengine/nuclei-templates/nuclei-templates` — a non-existent path.

The `getFileContents` API view (`api/views.py` line 1939) then returned:
```json
{"status": false, "message": "Invalid Path!"}
```

The SweetAlert2 JS handler displayed this as the "Error! Invalid Path" modal.

## Fix Applied

**File**: `scanEngine/templates/scanEngine/settings/tool.html` — line 105

```html
<!-- BEFORE (broken) -->
<span onclick="load_nuclei_template('{{file.3}}')">{{file.3}}</span>

<!-- AFTER (fixed) -->
<span onclick="load_nuclei_template('{{file.4}}')">{{file.4}}</span>
```

## Manual Application

```bash
# On the reNgine server, edit as root/rengine:
FILE="/home/rengine/web/scanEngine/templates/scanEngine/settings/tool.html"
python3 -c "
with open('$FILE', 'r') as f:
    lines = f.readlines()
lines[104] = lines[104].replace('file.3', 'file.4')
with open('$FILE', 'w') as f:
    f.writelines(lines)
print('Done')
"
# Then restart the web container:
docker restart rengine-web-1
```

## Verification

After the fix:
- All 6 custom templates resolve correctly (ssrf_nagli.yaml, crlf-param-injection.yaml, etc.)
- Clicking a template button opens the YAML viewer modal successfully
- No more "Error! Invalid Path" on the Nuclei tab
